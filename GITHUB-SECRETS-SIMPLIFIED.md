# GitHub Secrets Setup - Simplified Guide

## ✅ Only 2 Secrets Required!

After refactoring, you only need to add **2 GitHub secrets** (AWS credentials). Everything else is configured in `terraform.tfvars`.

---

## 🔐 Required GitHub Secrets

Go to: `https://github.com/nzeremc/accumulator/settings/secrets/actions`

### 1. AWS_ACCESS_KEY_ID
**Value**: Your AWS IAM access key ID
```
Example: AKIAIOSFODNN7EXAMPLE
```

### 2. AWS_SECRET_ACCESS_KEY
**Value**: Your AWS IAM secret access key
```
Example: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
```

---

## 📝 Configuration in terraform.tfvars

All other configuration is in your `terraform.tfvars` file:

```hcl
# General Configuration
aws_region   = "ap-south-1"  # Your AWS region
project_name = "docmp"
environment  = "production"

# Terraform Backend Configuration
terraform_state_bucket = "docmp-terraform-state-2101"  # Your S3 bucket

# ... rest of your configuration
```

## 🚀 How It Works Now

### 1. RDS Password
- **Auto-generated** using Terraform `random_password` resource
- **Stored** in AWS Secrets Manager automatically
- **16 characters** with special characters
- **No manual password needed**

### 2. AWS Region & S3 Bucket
- **Configured** in `terraform.tfvars`
- **Extracted** by GitHub Actions workflow
- **Used** for backend initialization and AWS operations

### 3. ECR Repository Names
- **Auto-generated** from `project_name` variable
- Creates: `{project_name}-app` and `{project_name}-app-db-init`
- Example: `docmp-app` and `docmp-app-db-init`

## 📋 Quick Setup Steps

### Step 1: Create AWS IAM Access Keys

1. Go to AWS Console → IAM → Users
2. Create user: `github-actions-terraform`
3. Attach policy from `terraform-iam-policy.json`
4. Create access key
5. **Save** Access Key ID and Secret Access Key

### Step 2: Add GitHub Secrets

```bash
# Using GitHub CLI
gh secret set AWS_ACCESS_KEY_ID
# Paste your access key ID

gh secret set AWS_SECRET_ACCESS_KEY
# Paste your secret access key
```

Or use the web interface:
1. Go to repository Settings → Secrets and variables → Actions
2. Click "New repository secret"
3. Add both secrets

### Step 3: Create terraform.tfvars

```bash
# Copy the example file
cp terraform.tfvars.example terraform.tfvars

# Edit with your values
# Make sure to set:
# - aws_region
# - terraform_state_bucket
# - other configuration values
```

### Step 4: Commit and Push

```bash
# terraform.tfvars is in .gitignore, so it won't be committed
git add .
git commit -m "Configure infrastructure"
git push origin main
```

## 🔍 How to Get RDS Password

After Terraform creates the infrastructure:

### Option 1: AWS Console
1. Go to AWS Secrets Manager
2. Find secret: `docmp-rds-master-password-XXXXX`
3. Click "Retrieve secret value"
4. View the password

### Option 2: AWS CLI
```bash
# List secrets
aws secretsmanager list-secrets --region ap-south-1

# Get secret value
aws secretsmanager get-secret-value \
  --secret-id docmp-rds-master-password-XXXXX \
  --region ap-south-1 \
  --query SecretString \
  --output text | jq .
```

### Option 3: Terraform Output
```bash
# After terraform apply
terraform output rds_master_password_secret_name
```

4. **Enable MFA** - For IAM users
5. **Monitor access** - Check CloudTrail logs

## 🐛 Troubleshooting

### Pipeline fails with "terraform.tfvars not found"
**Solution**: Make sure terraform.tfvars exists in repository root (but not committed to Git)

### Can't access RDS password
**Solution**: Check AWS Secrets Manager in your region for secret starting with `docmp-rds-master-password-`

### ECR repository not found
**Solution**: Terraform creates ECR repos automatically. Check they exist: `{project_name}-app` and `{project_name}-app-db-init`
